# 🎯 Railway Deploy - Ultra Simple (No CLI)

## What You Need

1. GitHub account with your repo pushed ✅
2. Railway account ✅
3. MongoDB URI ready: `mongodb+srv://tarshanvivek_db_user:[REDACTED]@cluster0.53oamx1.mongodb.net/tesla-shop?retryWrites=true&w=majority&appName=Cluster0`

## Deploy in 10 Minutes

### STEP 1: Go to Railway Dashboard

**Link:** https://railway.app/dashboard

Login if needed.

### STEP 2: Create New Project

1. Click blue **"New Project"** button (top right)
2. Select **"Deploy from GitHub repo"**
3. Click **"Configure GitHub App"** if needed
4. Select your repo: **`Crazy-Tarsh/Tesla-Tech-store`**
5. Click **"Deploy"**

Railway auto-detects and asks which Dockerfile to use.

### STEP 3: Select Server Dockerfile

1. When prompted for Dockerfile, select **`Dockerfile.server`**
2. Click **"Deploy"**
3. **WAIT** - Build starts (you see logs)

⏳ Takes 5-10 minutes. Don't close browser.

### STEP 4: Add Server Environment Variables

While server is building:

1. Click on **`tesla-server`** service in left panel
2. Click **"Variables"** tab
3. Click **"+ New Variable"**
4. Add each one:

**Variable 1:**
```
Key: MONGODB_URI
Value: mongodb+srv://tarshanvivek_db_user:[REDACTED]@cluster0.53oamx1.mongodb.net/tesla-shop?retryWrites=true&w=majority&appName=Cluster0
```
Click "Add"

**Variable 2:**
```
Key: JWT_SECRET
Value: your-super-secret-key-minimum-32-characters-long
```
Click "Add"

**Variable 3:**
```
Key: NODE_ENV
Value: production
```
Click "Add"

**Variable 4:**
```
Key: PORT
Value: 5000
```
Click "Add"

### STEP 5: Wait for Server Build

Watch the **"Logs"** tab. You should see:
- `Building...` (progress bar)
- `npm install` running
- Build completing
- Finally: ✅ **Running** (green status)

⏳ If still building, wait. This is normal for first deploy.

### STEP 6: Add Client Service

1. Click **"New Service"** button
2. Select **"Deploy from GitHub repo"**
3. Same repo: **`Crazy-Tarsh/Tesla-Tech-store`**
4. When asked for Dockerfile: **`Dockerfile.client`**
5. Click **"Deploy"**

### STEP 7: Add Client Environment Variables

While client builds:

1. Click on **`tesla-client`** service
2. Click **"Variables"** tab
3. Add one variable:

```
Key: VITE_API_URL
Value: https://tesla-server-xxxxx.up.railway.app
```

⚠️ Replace `xxxxx` with actual server URL (see next step)

### STEP 8: Get Your Live URLs

1. Click on **`tesla-server`** service
2. Look at top - should show:
   ```
   Domain
   https://tesla-server-prod-xxxxx.up.railway.app
   ```
3. Copy this URL
4. Go back to **`tesla-client`** Variables
5. Update `VITE_API_URL` with the actual server URL
6. Client auto-redeploys

### STEP 9: Test Your App

When BOTH services show ✅ **Running**:

1. **Open Server URL** in new browser tab:
   ```
   https://tesla-server-xxxxx.up.railway.app/api/health
   ```
   Should show:
   ```json
   {"status":"ok","name":"TESLA API",...}
   ```

2. **Open Client URL** in new browser tab:
   ```
   https://tesla-client-xxxxx.up.railway.app
   ```
   Should show: TESLA marketplace homepage

### ✅ DONE! Your app is live! 🎉

## If Build Fails

### Issue 1: Build keeps stuck on "Building..."
- Wait 15 minutes (first build is slow)
- If still stuck, click **"Redeploy"** button

### Issue 2: Build shows error "npm ERR!"
- Check **"Logs"** tab for exact error
- Usually missing env variables
- Add them to **"Variables"** tab
- Click **"Redeploy"**

### Issue 3: Service shows "Failed"
1. Click service name
2. Go to **"Logs"** tab
3. Read the error message
4. Fix based on error type:

**Error: "MONGODB_URI is required"**
→ Add MONGODB_URI to Variables

**Error: "Cannot find module"**
→ Check package.json is correct

**Error: "Port already in use"**
→ Set PORT to 5000 (server) or 80 (client)

### Issue 4: Client can't reach server
1. Go to **`tesla-client`** → Variables
2. Check `VITE_API_URL` matches actual server URL
3. Click **"Redeploy"**

## URL Format

Your URLs will look like:
```
https://tesla-server-prod-abc123def456.up.railway.app
https://tesla-client-prod-xyz789uvw012.up.railway.app
```

The random part (`abc123def456`) is auto-generated by Railway.

3. Service → **"Settings"** tab (change config)
